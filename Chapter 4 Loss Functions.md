## 第4章　損失関数

### 4.1　予測がどれくらい間違っているかを測る

機械学習では、モデルが入力に対して予測を出します。

たとえば、家の価格を予測するモデルなら、

```
入力：広さ80平米、築10年、駅徒歩8分
予測：7,500万円
```

のような出力を返します。

しかし、学習のためには、予測を出すだけでは足りません。

その予測がどれくらい正解からズレているのかを測る必要があります。

たとえば、実際の価格が8,000万円だったとします。

```
予測：7,500万円
正解：8,000万円
```

この場合、予測は500万円低かったことになります。

別の例を考えます。

```
予測：4,000万円
正解：8,000万円
```

この場合は、4,000万円もズレています。先ほどよりずっと悪い予測です。

人間が見れば、「500万円のズレより、4,000万円のズレの方が悪い」とすぐにわかります。しかし、機械学習では、この「どれくらい悪いか」を数値として定義する必要があります。

なぜなら、モデルの学習は、その数値を小さくするように進むからです。

この「予測がどれくらい間違っているか」を表す数値を、損失と呼びます。

そして、損失を計算するための関数を、損失関数と呼びます。

```
損失 = 損失関数(予測, 正解)
```

損失が大きいほど、予測は悪い。  
損失が小さいほど、予測はよい。

これが基本です。

学習とは、モデルの予測によって発生する損失を小さくすることです。

この考え方は、機械学習全体で非常に重要です。ニューラルネットワークでも、Transformer でも、大規模言語モデルでも同じです。

モデルが予測を出す。  
正解と比較する。  
損失を計算する。  
損失が小さくなるようにパラメータを更新する。

この流れが、機械学習の中心にあります。

### 4.2　損失関数とは何か

損失関数とは、モデルの予測と正解のズレを数値化する関数です。

たとえば、あるモデルが家の価格を予測するとします。

```
予測価格 = 7,500万円
正解価格 = 8,000万円
```

このとき、単純に差を取れば、

```
予測 - 正解 = -500万円
```

です。

しかし、このままだと少し扱いにくいです。

なぜなら、予測が正解より低い場合はマイナスになり、予測が正解より高い場合はプラスになるからです。

たとえば、

```
予測：7,500万円
正解：8,000万円
差：-500万円
```

と、

```
予測：8,500万円
正解：8,000万円
差：+500万円
```

は、どちらも500万円ズレています。

ズレの方向は違いますが、間違いの大きさとしては同じです。

そのため、損失関数では、ズレの向きではなく、ズレの大きさを扱うことが多いです。

単純な方法は、絶対値を取ることです。

```
損失 = |予測 - 正解|
```

これなら、低く外しても高く外しても、ズレの大きさを正の値として扱えます。

もう一つよく使われる方法は、差を二乗することです。

```
損失 = (予測 - 正解)^2
```

差を二乗すれば、マイナスの値もプラスになります。

このように、損失関数は「何をどれくらい悪いとみなすか」を定義します。

ここが重要です。

損失関数は単なる計算式ではありません。
モデルに対して、「こういう間違いを減らせ」と指示するものです。

家の価格予測で、価格のズレを損失にするなら、モデルは価格のズレを小さくするように学習します。

分類問題で、正解カテゴリの確率が低いことを損失にするなら、モデルは正解カテゴリに高い確率を出すように学習します。

言語モデルで、次の正解トークンの確率が低いことを損失にするなら、モデルは文脈に続く正しいトークンに高い確率を出すように学習します。

つまり、損失関数は、モデルが何を最適化するかを決める中心的な部品です。

### 4.3　平均二乗誤差

回帰問題でよく使われる損失関数に、平均二乗誤差があります。

英語では Mean Squared Error と呼ばれ、MSE と略されます。

まず、二乗誤差から考えます。

あるデータに対して、予測値と正解値があるとします。

```
予測 = 7,500
正解 = 8,000
```

この差は、

```
予測 - 正解 = -500
```

です。

この差を二乗します。

```
(-500)^2 = 250,000
```

これが二乗誤差です。

差を二乗することで、ズレがマイナスでもプラスの値になります。

複数のデータがある場合、それぞれについて二乗誤差を計算し、その平均を取ります。

```
データ1：予測 7,500、正解 8,000、二乗誤差 250,000
データ2：予測 5,200、正解 5,000、二乗誤差 40,000
データ3：予測 9,000、正解 10,000、二乗誤差 1,000,000
```

これらの平均を取ると、

```
平均二乗誤差 = (250,000 + 40,000 + 1,000,000) / 3
```

となります。

この値が小さいほど、モデルの予測は全体として正解に近いと考えます。

平均二乗誤差の特徴は、大きな誤差を強く罰することです。

たとえば、誤差が100の場合、二乗誤差は10,000です。  
誤差が1,000の場合、二乗誤差は1,000,000です。

誤差は10倍ですが、二乗誤差は100倍になります。

つまり、平均二乗誤差を使うと、「少しのズレ」よりも「大きなズレ」をかなり強く嫌うモデルになります。

これは多くの場合、都合がよい性質です。

たとえば、家の価格予測で、100万円のズレは許容できても、5,000万円のズレはかなり困るでしょう。平均二乗誤差は、このような大きなズレを強く減らそうとします。

一方で、外れ値に弱いという欠点もあります。

たとえば、ほとんどの物件は普通の価格帯なのに、ひとつだけ極端に高い豪邸が混ざっていたとします。そのデータに対する誤差が大きいと、二乗によって損失が非常に大きくなります。その結果、モデルがその外れ値に引っ張られすぎることがあります。

このように、損失関数には性質があります。

どの損失関数を使うかによって、モデルがどのような間違いを避けようとするかが変わります。

### 4.4　絶対誤差

平均二乗誤差と似た考え方に、絶対誤差があります。

絶対誤差は、予測と正解の差の絶対値を取ります。

```
絶対誤差 = |予測 - 正解|
```

たとえば、

```
予測 = 7,500
正解 = 8,000
```

なら、

```
絶対誤差 = |7,500 - 8,000|
         = 500
```

です。

予測が高すぎる場合でも同じです。

```
予測 = 8,500
正解 = 8,000
絶対誤差 = |8,500 - 8,000|
         = 500
```

絶対誤差では、ズレの向きは無視され、ズレの大きさだけを見ます。

複数のデータに対して平均を取ったものを、平均絶対誤差と呼びます。英語では Mean Absolute Error、MAE と呼ばれます。

平均二乗誤差と平均絶対誤差は、どちらも回帰問題で使われます。

違いは、大きな誤差をどれくらい強く罰するかです。

平均二乗誤差は、誤差を二乗するので、大きな誤差を非常に強く罰します。

平均絶対誤差は、誤差をそのまま大きさとして扱うので、大きな誤差への罰は比較的穏やかです。

たとえば、誤差が10倍になると、絶対誤差も10倍になります。しかし、二乗誤差では100倍になります。

そのため、外れ値の影響を抑えたい場合には、平均絶対誤差の方が扱いやすいことがあります。

ただし、ニューラルネットワークの学習では、損失関数を微分してパラメータ更新に使います。その観点では、二乗誤差の方が数学的に扱いやすい場面もあります。

ここで大事なのは、どちらが常に正しいという話ではないことです。

平均二乗誤差を使うと、大きな間違いを強く避けるモデルになります。  
平均絶対誤差を使うと、外れ値に比較的引っ張られにくいモデルになります。

損失関数の選択は、「何を悪い予測とみなすか」という設計判断です。

### 4.5　分類問題における損失

ここまでは、家の価格のような数値を予測する回帰問題を見てきました。

次に、分類問題を考えます。

分類問題では、出力はカテゴリです。

たとえば、犬と猫を分類するモデルなら、出力は「犬」か「猫」です。

```
入力：画像
出力：犬 / 猫
```

しかし、機械学習モデルは多くの場合、いきなり「犬」とだけ答えるのではなく、それぞれのカテゴリに対する確率を出します。

たとえば、

```
犬：0.90
猫：0.10
```

という出力なら、モデルは「犬である可能性が高い」と判断しています。

別の画像では、

```
犬：0.40
猫：0.60
```

かもしれません。

この場合は、猫の方が少し高いですが、モデルはあまり自信がないとも言えます。

分類問題で学習するには、この確率分布と正解ラベルを比較する必要があります。

たとえば、正解が犬だったとします。

モデルの出力が、

```
犬：0.90
猫：0.10
```

なら、よい予測です。

一方、

```
犬：0.10
猫：0.90
```

なら、悪い予測です。

では、この「よい」「悪い」をどう数値化するのでしょうか。

分類問題でよく使われるのが、交差エントロピーという損失関数です。

交差エントロピーは、直感的には「正解カテゴリにどれくらい高い確率を割り当てたか」を見る損失です。

正解カテゴリに高い確率を割り当てていれば、損失は小さくなります。  
正解カテゴリに低い確率しか割り当てていなければ、損失は大きくなります。

つまり、分類問題における学習は、正解カテゴリの確率を高くするようにモデルを調整することだと考えられます。

### 4.6　確率として出力する

分類モデルが確率を出すという考え方を、もう少し丁寧に見ます。

犬、猫、鳥の3種類を分類するモデルがあるとします。

ある画像を入力したとき、モデルは次のような出力を返すかもしれません。

```
犬：0.70
猫：0.20
鳥：0.10
```

この3つの値は、合計すると1になります。

```
0.70 + 0.20 + 0.10 = 1.00
```

このように、複数の候補に対して、合計が1になるような数値を割り当てたものを、確率分布と呼びます。

分類問題では、モデルの出力を確率分布として扱うことがよくあります。

ただし、ニューラルネットワークの内部では、最初からきれいな確率が出てくるわけではありません。

実際には、モデルはまず各カテゴリに対応するスコアを出します。

```
犬：3.2
猫：1.9
鳥：0.5
```

この段階では、値は確率ではありません。合計しても1になりませんし、マイナスになることもあります。

そこで、これらのスコアを確率に変換します。

この変換に使われる代表的な関数が softmax です。

softmax は、任意のスコアの列を、合計が1になる確率分布に変換します。

```
スコア
犬：3.2
猫：1.9
鳥：0.5

softmax後
犬：0.75
猫：0.20
鳥：0.05
```

softmax の重要な性質は、スコアが高いカテゴリほど高い確率になることです。

また、すべての確率は0以上で、合計は1になります。

Transformer や大規模言語モデルでも、最後の出力では softmax が使われます。

言語モデルの場合、カテゴリは「語彙中のトークン」です。

たとえば語彙に5万個のトークンがあるなら、モデルは次に来るトークンについて、5万個の候補それぞれにスコアを出します。

そのスコアを softmax で確率分布に変換します。

```
猫：0.12
犬：0.04
で：0.03
は：0.02
...
```

そして、正解トークンに高い確率を出せるように学習します。

つまり、分類問題の考え方は、言語モデルにも直接つながります。

言語モデルは、「次のトークンを語彙全体の中から分類する問題」と見ることができます。

### 4.7　交差エントロピー

分類問題で非常によく使われる損失関数が、交差エントロピーです。

名前は難しく見えますが、最初は直感で理解すれば十分です。

交差エントロピーは、正解に対してモデルがどれくらい低い確率を出してしまったかを罰する損失です。

たとえば、犬と猫を分類する問題で、正解が犬だったとします。

モデルAは、次のように出力しました。

```
犬：0.90
猫：0.10
```

モデルBは、次のように出力しました。

```
犬：0.55
猫：0.45
```

モデルCは、次のように出力しました。

```
犬：0.10
猫：0.90
```

正解は犬なので、犬に高い確率を出しているほどよい予測です。

この場合、モデルAはよい予測です。モデルBは一応犬の方が高いですが、自信が弱いです。モデルCはかなり悪い予測です。

交差エントロピーでは、正解カテゴリの確率が高いほど損失が小さくなります。

逆に、正解カテゴリの確率が低いほど損失が大きくなります。

数式で書くと、正解カテゴリに対する予測確率を p としたとき、損失はおおよそ次のようになります。

```
損失 = -log(p)
```

ここで log は対数です。

対数に慣れていなくても、直感としては次のように見ればよいです。

```
正解確率 p が高い → -log(p) は小さい
正解確率 p が低い → -log(p) は大きい
```

たとえば、正解カテゴリに0.90の確率を出していれば、損失は小さくなります。  
正解カテゴリに0.10の確率しか出していなければ、損失は大きくなります。  
正解カテゴリに0.001の確率しか出していなければ、損失は非常に大きくなります。

この性質は重要です。

分類問題では、単に正解カテゴリが一番高ければよい、というだけではありません。

正解にどれくらい高い確率を割り当てたかも重要です。

たとえば、正解が犬のとき、

```
犬：0.51
猫：0.49
```

でも分類結果としては犬です。

しかし、モデルはかなり迷っています。

一方、

```
犬：0.99
猫：0.01
```

なら、モデルは強く犬だと判断しています。

交差エントロピーは、この違いを反映します。

正解に高い確率を出すほど、損失は小さくなります。

このため、交差エントロピーを使って学習すると、モデルは正解カテゴリに高い確率を出すように調整されます。

### 4.8　損失が小さいほどよい、とはどういうことか

機械学習では、「損失を小さくする」とよく言います。

しかし、損失が小さいとは、具体的にどういう意味でしょうか。

回帰問題では、損失が小さいということは、予測値が正解値に近いということです。

たとえば、家の価格予測で、

```
正解：8,000万円
予測：7,950万円
```

なら、ズレは小さいです。

一方、

```
正解：8,000万円
予測：3,000万円
```

なら、ズレは大きいです。

したがって、損失も大きくなります。

分類問題では、損失が小さいということは、正解カテゴリに高い確率を出しているということです。

たとえば、正解が「猫」のとき、

```
猫：0.95
犬：0.03
鳥：0.02
```

なら、損失は小さいです。

一方、

```
猫：0.02
犬：0.90
鳥：0.08
```

なら、損失は大きいです。

言語モデルでも同じです。

入力が、

```
吾輩は
```

で、正解の次トークンが、

```
猫
```

だったとします。

モデルが、

```
猫：0.70
犬：0.10
人間：0.05
...
```

と出せば、損失は比較的小さいです。

一方、

```
猫：0.001
犬：0.40
昨日：0.20
...
```

と出せば、損失は大きくなります。

つまり、損失が小さいとは、モデルが正解に近い予測をしているということです。

ただし、ここで注意すべきことがあります。

損失が小さいというのは、あくまで「その損失関数から見てよい」という意味です。

もし損失関数の設計が悪ければ、損失が小さくても、人間にとって望ましいモデルになるとは限りません。

たとえば、言語モデルで「次のトークン予測」の損失を小さくすると、文章の続きを予測する能力は高まります。しかし、それだけで必ずしも「正直である」「安全である」「ユーザーの意図に従う」といった性質が十分に得られるわけではありません。

そのため、現代のAIでは、事前学習の損失だけでなく、指示への追従性、安全性、人間の好みなどを反映する追加の学習が行われます。

損失関数は、モデルに何を学ばせるかを決める重要な設計です。

### 4.9　損失関数を選ぶ意味

損失関数を選ぶことは、「何を間違いとみなすか」を決めることです。

これは単なる技術的な選択ではありません。

モデルの振る舞いを決める設計判断です。

たとえば、家の価格予測で平均二乗誤差を使うと、大きな誤差を強く減らそうとします。

これは、「大きく外すことを特に避けたい」という方針に近いです。

一方、平均絶対誤差を使うと、外れ値に対して比較的穏やかになります。

これは、「極端なデータに引っ張られすぎないようにしたい」という方針に近いです。

分類問題で交差エントロピーを使うと、正解カテゴリに高い確率を出すように学習します。

これは、「正解に対する確信度を高めたい」という方針に近いです。

このように、損失関数はモデルに対する評価基準です。

モデルは、その評価基準をよくするように学習します。

したがって、評価基準がズレていると、モデルの学習もズレます。

たとえば、あるサービスで「クリック率」だけを最大化するモデルを作ったとします。

すると、モデルはユーザーがクリックしやすいものを優先的に出すようになります。

しかし、クリックされやすいものが、必ずしもユーザーにとって本当に価値があるとは限りません。刺激的なタイトルや、不安を煽る内容がクリックされやすい場合、モデルはそうしたものを優先してしまうかもしれません。

つまり、最適化する指標を間違えると、望ましくない結果が出ることがあります。

機械学習では、モデルは「損失関数で定義された目的」に従って改善されます。

だからこそ、損失関数や評価指標の設計は重要です。

Transformer のようなモデルでも同じです。

次トークン予測の損失を小さくすれば、文章の統計的な予測能力は高まります。しかし、それだけで人間にとって理想的な応答をするとは限りません。

この問題意識が、後の instruction tuning や RLHF などにつながります。

### 4.10　学習とは損失を小さくすることである

ここまで見てきたように、機械学習の学習は、損失を小さくする作業として理解できます。

学習の流れをもう一度整理します。

まず、モデルに入力を与えます。

```
入力 → モデル
```

モデルは予測を出します。

```
モデル → 予測
```

その予測と正解を比較します。

```
予測 vs 正解
```

損失関数を使って、どれくらい間違っているかを数値化します。

```
損失 = 損失関数(予測, 正解)
```

そして、この損失が小さくなるように、モデルのパラメータを更新します。

```
パラメータを少し調整する
```

この処理を大量のデータに対して何度も繰り返します。

すると、モデルはだんだん損失が小さくなるようなパラメータを持つようになります。

この意味で、機械学習は「損失関数を最小化する問題」と見ることができます。

より数学的に言えば、モデルのパラメータを変数として、損失関数の値ができるだけ小さくなるようなパラメータを探します。

しかし、最初から数学的に厳密に考える必要はありません。

直感としては、

```
モデルが間違える
↓
間違いの大きさを測る
↓
間違いが減る方向にモデルを少し直す
↓
これを何度も繰り返す
```

という理解で十分です。

この「間違いが減る方向にモデルを少し直す」方法が、次の章で扱う勾配降下法です。

損失関数は、モデルが今どれくらい悪いかを教えてくれます。
勾配降下法は、損失を減らすためにパラメータをどちらへ動かせばよいかを教えてくれます。

この2つがつながると、機械学習の学習プロセスがかなり見えてきます。

### 4.11　1つのデータの損失と、全体の損失

損失には、1つのデータに対する損失と、データ全体に対する損失があります。

たとえば、家の価格予測で、1件の物件について、

```
予測：7,500万円
正解：8,000万円
```

だったとします。

この1件について損失を計算できます。

しかし、モデルを評価するときには、1件だけを見ても十分ではありません。

ある1件に対しては正確でも、他の物件では大きく外すかもしれません。

そのため、通常は多くのデータに対する損失をまとめて考えます。

```
データ1の損失
データ2の損失
データ3の損失
...
```

これらを平均したものを、全体の損失として扱うことが多いです。

```
全体の損失 = 各データの損失の平均
```

学習では、この全体の損失を小さくすることを目指します。

ただし、実際の学習では、すべての訓練データを毎回使うとは限りません。

大規模なデータセットでは、データが何百万、何千万、何億件もあることがあります。毎回すべてのデータで損失を計算すると、計算量が大きすぎます。

そこで、訓練データの一部を取り出して、その範囲で損失を計算し、パラメータを更新します。

この一部のデータのまとまりをミニバッチと呼びます。

たとえば、訓練データ全体から64件だけ取り出して、その64件の平均損失を計算します。そして、その損失が小さくなるようにパラメータを更新します。

次に、また別の64件を使います。

このように、一部のデータを使って少しずつ学習を進めるのが、現代のニューラルネットワークでは一般的です。

この話は、次の章の「確率的勾配降下法」や「ミニバッチ学習」で詳しく扱います。

ここでは、損失には「1つのデータに対する損失」と「多くのデータに対する平均的な損失」がある、と理解しておけば十分です。

### 4.12　訓練損失と検証損失

損失を見るときには、「どのデータに対する損失か」が重要です。

学習に使っている訓練データに対する損失を、訓練損失と呼びます。

一方、学習には使っていない検証データに対する損失を、検証損失と呼びます。

この区別は非常に重要です。

モデルは、訓練データに対する損失を小さくするように学習します。したがって、学習が進むと、訓練損失は下がっていくことが多いです。

しかし、訓練損失が下がることと、未知のデータに強くなることは同じではありません。

モデルが訓練データを丸暗記してしまえば、訓練損失は非常に小さくなるかもしれません。しかし、未知のデータではうまく予測できない可能性があります。

この状態が過学習です。

過学習が起きると、訓練損失は下がっているのに、検証損失は下がらなくなったり、逆に上がったりします。

```
訓練損失：下がり続ける
検証損失：途中から上がる
```

このような場合、モデルは訓練データに適応しすぎて、一般的な規則性ではなく、訓練データ固有の細かい特徴まで覚えてしまっている可能性があります。

したがって、機械学習では、訓練損失だけでなく、検証損失を見ることが重要です。

モデルの目的は、訓練データでよい成績を出すことではありません。未知のデータに対してよい予測をすることです。

そのため、損失を見るときには常に、

```
訓練データでの損失なのか
検証データでの損失なのか
テストデータでの損失なのか
```

を区別する必要があります。

この話は、第6章の「訓練データ、検証データ、テストデータ」と、第7章の「過学習と汎化」でさらに詳しく扱います。

### 4.13　言語モデルにおける損失

ここまでの話を、言語モデルに接続してみます。

言語モデルの基本的な学習では、文脈から次のトークンを予測します。

たとえば、次の文章があるとします。

```
吾輩は猫である
```

これを、次のような学習データとして見ることができます。

```
入力：吾輩は
正解：猫
入力：吾輩は猫
正解：で
入力：吾輩は猫で
正解：ある
```

モデルは、入力された文脈に対して、次のトークンの確率分布を出します。

たとえば、

```
入力：吾輩は
```

に対して、モデルが次のような確率を出したとします。

```
猫：0.60
犬：0.10
人間：0.05
学生：0.03
...
```

正解が「猫」なら、これは比較的よい予測です。

一方、

```
猫：0.001
犬：0.40
昨日：0.20
...
```

のような出力なら、かなり悪い予測です。

このとき、交差エントロピーを使って損失を計算します。

正解トークンである「猫」に高い確率を出していれば、損失は小さくなります。  
正解トークンである「猫」に低い確率しか出していなければ、損失は大きくなります。

つまり、言語モデルの事前学習とは、大量のテキストに対して、次のトークン予測の損失を小さくすることです。

この見方は非常に重要です。

言語モデルは、一見すると文章を理解しているように見えます。実際、高度な言語モデルは非常に複雑な能力を示します。

しかし、基本的な学習目的は、

```
文脈を見て、次のトークンを予測する
```

です。

そして、その予測が正解トークンに近づくように、交差エントロピー損失を小さくします。

Transformer の論文や LLM の解説で「loss」や「cross entropy loss」という言葉が出てきたら、この話を思い出してください。

それは、モデルが次の正解トークンにどれくらい高い確率を出せているかを測っているのです。

### 4.14　損失と精度は同じではない

分類問題では、「精度」という指標もよく使われます。

精度とは、全体のうち、どれくらい正しく分類できたかを表す指標です。

たとえば、100枚の画像を分類して、90枚を正しく分類できたなら、精度は90%です。

```
精度 = 正解数 / 全体数
```

これは直感的でわかりやすい指標です。

しかし、損失と精度は同じではありません。

たとえば、正解が犬の画像に対して、モデルAが次のように出力したとします。

```
犬：0.51
猫：0.49
```

モデルBは次のように出力しました。

```
犬：0.99
猫：0.01
```

どちらも、最終的な分類結果は犬です。つまり、精度だけを見ると、どちらも正解です。

しかし、モデルAはかなり迷っています。モデルBは強く犬だと判断しています。

交差エントロピー損失では、モデルBの方が損失は小さくなります。

逆に、正解が犬のとき、

```
犬：0.49
猫：0.51
```

なら分類結果は猫なので不正解です。

```
犬：0.01
猫：0.99
```

も不正解です。

精度だけを見ると、どちらも同じ不正解です。

しかし、損失では後者の方がずっと悪くなります。なぜなら、正解である犬にほとんど確率を割り当てていないからです。

このように、精度は「最終的に当たったか外れたか」を見る指標です。

一方、損失は「正解にどれくらい確率を割り当てたか」「どれくらいズレていたか」をより細かく見ます。

学習では、精度そのものではなく、損失関数を使ってパラメータを更新することが一般的です。

なぜなら、損失は連続的に変化し、微分しやすいからです。

精度は、当たりか外れかの離散的な値です。少しだけパラメータを変えても、精度が変わらないことが多く、学習の方向を知るには使いにくいです。

そのため、学習には損失関数を使い、評価には精度やF値などの指標を併用する、という形がよく使われます。

### 4.15　本章のまとめ

この章では、損失関数について学びました。

損失関数とは、モデルの予測と正解のズレを数値化する関数です。

```
損失 = 損失関数(予測, 正解)
```

損失が大きいほど、予測は悪い。  
損失が小さいほど、予測はよい。

機械学習の学習とは、この損失を小さくするように、モデルのパラメータを調整することです。

回帰問題では、平均二乗誤差や平均絶対誤差がよく使われます。

平均二乗誤差は、大きな誤差を強く罰します。

```
損失 = (予測 - 正解)^2
```

分類問題では、交差エントロピーがよく使われます。

交差エントロピーは、正解カテゴリに高い確率を出していれば小さくなり、正解カテゴリに低い確率しか出していなければ大きくなります。

```
損失 = -log(正解カテゴリに割り当てた確率)
```

言語モデルでも、基本的には交差エントロピーが使われます。

入力された文脈に対して、次の正解トークンに高い確率を出せるように学習します。

この章で一番重要な考え方は、次の一文です。

損失関数とは、モデルにとって何が悪い予測かを数値で定義するものであり、学習とはその損失を小さくすることである。

Transformer を理解するためにも、この考え方は非常に重要です。

Transformer は複雑な構造を持っていますが、学習時には、やはり予測と正解を比較し、損失を計算し、その損失が小さくなるようにパラメータを更新しています。

次の章では、損失を小さくするために、どうやってパラメータを動かすのかを学びます。その中心になるのが、最適化と勾配降下法です。
