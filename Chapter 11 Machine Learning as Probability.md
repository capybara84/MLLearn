## 第11章　確率として見る機械学習

**この章でわかること**

- 機械学習で確率が使われる理由
- 確率分布、尤度、最大尤度推定の直感
- 交差エントロピーと尤度の関係
- 言語モデルが次トークンの条件付き確率を扱うこと

### 11.1　なぜ確率が出てくるのか

機械学習では、確率という考え方がよく出てきます。

分類問題では、モデルは各カテゴリに対する確率を出しました。

```text
犬：0.87
猫：0.13
```

スパム判定でも、モデルは次のような値を出します。

```text
スパムである確率：0.92
```

言語モデルでも、次に来るトークンの確率分布を出します。

```text
猫：0.35
犬：0.08
人間：0.04
学生：0.03
...
```

なぜ機械学習では確率が重要なのでしょうか。

理由は、機械学習が不確実な問題を扱うからです。

たとえば、画像を見て犬か猫かを判定する問題を考えます。

人間にとっては簡単に見える画像でも、モデルにとっては曖昧な場合があります。

画像が暗い。  
一部しか写っていない。  
犬と猫が似た姿勢をしている。  
画像がぼやけている。

このような場合、「絶対に犬である」と断言するよりも、「犬である可能性が高い」と表現する方が自然です。

スパム判定でも同じです。

あるメールが明らかにスパムなら、モデルは高いスパム確率を出すでしょう。

しかし、少し怪しいだけのメールなら、モデルは中間的な確率を出すかもしれません。

```text
明らかにスパム：0.99
少し怪しい：0.65
ほぼ正常：0.03
```

このように、確率はモデルの不確実性を表すために使われます。

言語モデルでは、さらに確率が中心的な役割を持ちます。

ある文脈に続くトークンは、一つに決まるわけではありません。

たとえば、

```text
今日はとても
```

の後には、さまざまなトークンが来る可能性があります。

```text
暑い
寒い
楽しい
忙しい
眠い
```

どれも文脈によっては自然です。

したがって、言語モデルは「次は絶対にこの単語」と決めるのではなく、候補ごとに確率を出します。

```text
暑い：0.30
寒い：0.18
楽しい：0.10
忙しい：0.08
眠い：0.05
...
```

この確率分布をもとに、次のトークンを選びます。

つまり、機械学習では、曖昧さや不確実性を扱うために確率が必要になります。

確率として出力することで、モデルは「どれくらいありそうか」を候補ごとに表せます。

```mermaid
flowchart LR
    A["入力"] --> B["モデル"]
    B --> C["候補ごとのスコア"]
    C --> D["確率分布"]
    D --> E["判断や生成に使う"]
```

### 11.2　不確実性を扱う

現実のデータには、ほとんどの場合、不確実性があります。

同じ入力に見えても、出力が完全には決まらないことがあります。

たとえば、商品の売上予測を考えます。

同じ商品、同じ価格、同じ曜日、同じ広告費でも、売上は毎回まったく同じにはなりません。

天気、競合商品の動き、SNSでの話題、ユーザーの気分、偶然の要因などが影響します。

つまり、入力が似ていても、出力には揺らぎがあります。

家の価格予測でも同じです。

広さ、築年数、駅距離、地域が同じような物件でも、実際の価格は完全には一致しません。

売主の事情、内装の状態、眺望、タイミング、買主の好みなどが影響します。

このような問題では、「ただ一つの正解値」を完全に当てることは難しいです。

モデルは、もっともありそうな値を予測したり、値の分布を予測したりします。

分類問題でも、不確実性があります。

ある画像が犬か猫か、モデルにとって判断が難しい場合があります。

そのとき、モデルが次のように出力するのは自然です。

```text
犬：0.52
猫：0.48
```

これは、モデルがかなり迷っていることを表します。

一方、次のような出力なら、モデルは強く犬だと判断しています。

```text
犬：0.99
猫：0.01
```

このように、確率は、単なる分類結果だけでは表せない「自信の度合い」を表せます。

ただし、注意が必要です。

モデルが出す確率は、必ずしも現実の確率と完全に一致するわけではありません。

モデルが「90%」と言っても、本当に同じようなケースで90%正解するとは限りません。

このズレは、モデルの calibration、つまり確率の較正の問題です。

機械学習では、確率を出すことは便利ですが、その確率がどれくらい信頼できるかも考える必要があります。

### 11.3　確率分布

確率分布とは、複数の可能性に対して、それぞれどれくらい起こりやすいかを割り当てたものです。

分類問題では、カテゴリごとに確率を割り当てます。

たとえば、犬・猫・鳥の分類なら、次のようになります。

```text
犬：0.70
猫：0.20
鳥：0.10
```

この場合、犬である可能性が最も高く、次に猫、最後に鳥です。

確率分布では、各確率は0以上で、合計は1になります。

```text
0.70 + 0.20 + 0.10 = 1.00
```

言語モデルでも、出力は確率分布です。

語彙が50,000トークンあるなら、モデルは50,000個のトークンに対して確率を割り当てます。

```text
猫：0.12
犬：0.04
で：0.03
は：0.02
今日：0.01
...
```

この確率をすべて足すと1になります。

モデルは、この確率分布を使って次のトークンを選びます。

確率分布は、モデルの出力候補全体を表します。

単に「一番ありそうな答え」だけでなく、「他にどんな可能性があるか」も表している点が重要です。

たとえば、次の2つの分布を比べてみます。

```text
分布A：
犬：0.90
猫：0.05
鳥：0.05
```

```text
分布B：
犬：0.40
猫：0.35
鳥：0.25
```

どちらも一番高いのは犬です。

したがって、最終的な予測カテゴリだけを見ると、どちらも「犬」です。

しかし、分布Aではモデルは強く犬だと判断しています。

分布Bでは、犬が一番高いものの、猫や鳥の可能性もかなりあります。

この違いは重要です。

確率分布を見ることで、モデルの迷い方や候補の広がりがわかります。

### 11.4　尤度

確率を使った機械学習では、「尤度」という考え方が重要になります。

尤度は、英語では likelihood と呼ばれます。

尤度とは、観測されたデータが、あるモデルのもとでどれくらいもっともらしいかを表す量です。

少し抽象的なので、例で考えます。

あるコインがあります。

このコインが表を出す確率を知りたいとします。

普通の公平なコインなら、表の確率は0.5です。

しかし、このコインは偏っているかもしれません。

そこで、実際に10回投げてみたところ、次の結果になりました。

```text
表、表、表、表、表、表、表、裏、表、表
```

表が9回、裏が1回です。

この結果を見ると、「表が出る確率は0.5」というモデルよりも、「表が出る確率は0.9」というモデルの方が、このデータを説明しやすそうです。

つまり、観測されたデータは、表確率0.9のモデルのもとでよりもっともらしい。

この「データのもっともらしさ」が尤度です。

機械学習でも同じように考えます。

モデルがデータに対して高い確率を割り当てるなら、そのデータはモデルのもとで尤もらしいと言えます。

たとえば、言語モデルで次の文があるとします。

```text
吾輩は猫である
```

モデルがこの文に含まれる各トークンを高い確率で予測できるなら、この文はモデルにとって尤もらしい文です。

一方、モデルが各トークンに低い確率しか出せないなら、その文はモデルにとってあまり尤もらしくありません。

尤度は、モデルが観測データをどれくらいうまく説明しているかを見るための考え方です。

### 11.5　最大尤度推定

最大尤度推定とは、観測されたデータの尤度が最大になるように、モデルのパラメータを決める方法です。

英語では Maximum Likelihood Estimation と呼ばれ、MLE と略されます。

考え方はシンプルです。

手元に観測データがあります。

そのデータが最も起こりやすくなるようなモデルのパラメータを選びます。

先ほどのコインの例で考えます。

10回投げて、表が9回、裏が1回出ました。

```text
表：9回
裏：1回
```

このデータを最もよく説明する表の確率は、直感的には0.9です。

表が90%出るコインだと考えれば、10回中9回表が出た結果はかなり自然です。

このように、観測されたデータが最ももっともらしくなるようにパラメータを選ぶのが最大尤度推定です。

機械学習でも、多くの学習は最大尤度推定として見ることができます。

たとえば、分類問題では、正解クラスに高い確率を割り当てるようにモデルを学習します。

正解が犬の画像に対して、

```text
犬：0.90
猫：0.10
```

と出すモデルは、正解データに高い確率を割り当てています。

一方、

```text
犬：0.10
猫：0.90
```

と出すモデルは、正解データに低い確率しか割り当てていません。

最大尤度の考え方では、正解データに高い確率を割り当てるモデルを良いモデルと見ます。

言語モデルでも同じです。

文脈に対して実際に続いたトークンに高い確率を出すように学習します。

```text
入力：吾輩は
正解：猫
```

このとき、「猫」に高い確率を出すモデルは、このデータをうまく説明しています。

「猫」に低い確率しか出さないモデルは、このデータをうまく説明していません。

つまり、次トークン予測の学習は、観測されたテキストの尤度を高める学習として見ることができます。

### 11.6　交差エントロピーとの関係

前の章で、分類問題では交差エントロピー損失を使うと学びました。

交差エントロピーは、正解クラスに低い確率を出すと大きくなり、正解クラスに高い確率を出すと小さくなります。

正解クラスに対する予測確率を `p` とすると、損失は次のように表せます。

```text
損失 = -log(p)
```

ここで、`p` が大きいほど損失は小さくなります。

```text
p が高い → 損失は小さい
p が低い → 損失は大きい
```

これは、最大尤度推定と深く関係しています。

最大尤度推定では、正解データに高い確率を割り当てるようにします。

つまり、正解の確率 `p` を大きくしたい。

一方、交差エントロピー損失では、`-log(p)` を小さくしたい。

`p` を大きくすることと、`-log(p)` を小さくすることは、同じ方向の目的です。

```text
正解の確率 p を大きくする
=
-log(p) を小さくする
```

そのため、交差エントロピー損失を最小化することは、正解データの尤度を最大化することと対応します。

厳密には、複数のデータに対しては、各データの確率を掛け合わせたものが尤度になります。

しかし、確率をたくさん掛けると非常に小さな値になり、扱いにくくなります。

そこで、対数を取ります。

確率の積は、対数を取ると和になります。

```text
確率を掛け合わせる
↓
対数を取る
↓
足し算として扱える
```

さらに、最大化問題を最小化問題として扱うために、マイナスを付けます。

この結果、負の対数尤度、つまり negative log likelihood という形になります。

分類で使う交差エントロピー損失は、この負の対数尤度と非常に近いものです。

つまり、交差エントロピーは単なる便利な損失関数ではありません。

「観測された正解データを、モデルが高い確率で説明できるようにする」という確率的な意味を持っています。

### 11.7　「もっともらしさ」を最大化する

最大尤度推定の考え方を直感的に言うと、モデルは「実際に観測されたデータがもっともらしく見えるように」学習します。

分類問題では、正解ラベルがもっともらしくなるようにします。

たとえば、正解が猫である画像に対して、モデルが次のように出力したとします。

```text
猫：0.95
犬：0.03
鳥：0.02
```

このモデルは、観測された正解「猫」を高い確率で説明しています。

一方、次のような出力ならどうでしょうか。

```text
猫：0.01
犬：0.90
鳥：0.09
```

このモデルは、観測された正解「猫」をほとんど説明できていません。

したがって、損失は大きくなります。

言語モデルでも同じです。

次の文章が学習データにあるとします。

```text
吾輩は猫である
```

モデルは、各位置で次のトークンを予測します。

```text
入力：吾輩は
正解：猫

入力：吾輩は猫
正解：で

入力：吾輩は猫で
正解：ある
```

このとき、モデルは実際に続いたトークンに高い確率を出すように学習します。

```text
「猫」に高い確率を出す
「で」に高い確率を出す
「ある」に高い確率を出す
```

これにより、学習データ中の文章がモデルにとってもっともらしくなります。

ただし、ここで注意が必要です。

モデルは「真実」を直接最大化しているわけではありません。

モデルは、学習データに現れたパターンの尤もらしさを高めています。

学習データに誤りが含まれていれば、その誤りも学習される可能性があります。

学習データに偏りがあれば、その偏りも反映される可能性があります。

つまり、最大尤度の学習は強力ですが、データの品質や偏りに大きく依存します。

### 11.8　言語モデルへのつながり

確率の考え方は、言語モデルを理解する上で非常に重要です。

言語モデルとは、簡単にいうと、トークン列に確率を割り当てるモデルです。

ある文がどれくらい自然か。  
ある文脈の次にどのトークンが来やすいか。

これを確率として扱います。

たとえば、次の文脈があります。

```text
今日はとても
```

この後に来るトークンとして、モデルは確率分布を出します。

```text
暑い：0.30
寒い：0.20
楽しい：0.08
忙しい：0.07
...
```

この確率分布は、言語モデルが学習データから得た言語パターンに基づいています。

学習時には、実際に続いたトークンに高い確率を出すようにモデルを更新します。

```text
入力：今日はとても
正解：暑い
```

もし正解が「暑い」なら、「暑い」の確率を高くするように学習します。

これを膨大なテキストに対して繰り返します。

その結果、モデルは文脈に応じた次トークンの確率分布を出せるようになります。

文章全体の確率も、次トークン確率の積として考えることができます。

たとえば、

```text
吾輩は猫である
```

という文の確率は、非常に単純化すると、次のように分解できます。

```text
P(吾輩は猫である)
=
P(吾輩)
× P(は | 吾輩)
× P(猫 | 吾輩は)
× P(で | 吾輩は猫)
× P(ある | 吾輩は猫で)
```

ここで `P(猫 | 吾輩は)` は、「吾輩は」という文脈の後に「猫」が来る確率です。

このように、言語モデルは、文脈付きの次トークン確率を学習します。

Transformer は、この次トークン確率を計算するための強力なモデル構造です。

### 11.9　条件付き確率

言語モデルを理解するには、条件付き確率の考え方が重要です。

条件付き確率とは、ある条件が与えられたときに、別の事象が起こる確率です。

記号では、次のように書きます。

```text
P(A | B)
```

これは、「B が与えられたときに A が起こる確率」という意味です。

たとえば、

```text
P(猫 | 吾輩は)
```

は、「吾輩は」という文脈が与えられたときに、次に「猫」が来る確率です。

日常的にも、条件付き確率は自然に使っています。

たとえば、何の文脈もなく「猫」という単語が出てくる確率を考えるのと、「吾輩は」の後に「猫」が出てくる確率を考えるのでは、まったく違います。

```text
P(猫)
P(猫 | 吾輩は)
```

後者の方が、「猫」が出てくる可能性はかなり高いでしょう。

言語モデルは、文脈を条件として、次のトークンの確率を出します。

```text
P(次のトークン | これまでのトークン列)
```

たとえば、

```text
P(暑い | 今日はとても)
P(寒い | 今日はとても)
P(楽しい | 今日はとても)
```

のような確率を計算します。

この条件付き確率をうまく推定できるほど、言語モデルは自然な文章を生成しやすくなります。

Transformer は、Self-Attention によって文脈中の各トークンの関係を扱い、その文脈に基づいて次トークンの条件付き確率を出します。

つまり、大規模言語モデルの中心には、条件付き確率の考え方があります。

### 11.10　確率分布からトークンを選ぶ

言語モデルは、次トークンの確率分布を出します。

しかし、確率分布を出すだけでは、まだ文章は生成されません。

実際に次のトークンを選ぶ必要があります。

たとえば、モデルが次のような確率分布を出したとします。

```text
暑い：0.30
寒い：0.20
楽しい：0.08
忙しい：0.07
眠い：0.05
...
```

最も単純な方法は、一番確率が高いトークンを選ぶことです。

この場合は「暑い」を選びます。

```text
最も高い確率のトークンを選ぶ
↓
暑い
```

この方法は greedy decoding と呼ばれます。

ただし、常に一番確率が高いトークンだけを選ぶと、文章が単調になったり、同じような表現になりやすかったりします。

そこで、確率分布に従ってランダムに選ぶ方法もあります。

確率が高いトークンほど選ばれやすいが、確率が低いトークンもたまには選ばれる、という方法です。

```text
暑い：選ばれやすい
寒い：そこそこ選ばれる
楽しい：たまに選ばれる
```

このようにすると、生成に多様性が出ます。

生成AIでよく聞く temperature や top-k、top-p といった設定は、このトークン選択の仕方に関係します。

temperature を高くすると、低確率の候補も選ばれやすくなり、出力が多様になります。

temperature を低くすると、高確率の候補が選ばれやすくなり、出力が安定します。

つまり、言語モデルの生成は、次の2段階に分けられます。

```text
1. モデルが次トークンの確率分布を出す
2. その確率分布から実際のトークンを選ぶ
```

この2つは別の処理です。

モデルが出す確率分布と、そこからどのように選ぶかを分けて考えることが重要です。

生成では、この処理を何度も繰り返して文章を伸ばしていきます。

```mermaid
flowchart LR
    A["これまでのトークン列"] --> B["言語モデル"]
    B --> C["次トークンの確率分布"]
    C --> D["選択方法"]
    D --> E["次トークン"]
    E --> F["文脈に追加"]
    F --> A
```

#### PyTorchで確認してみる

確率分布からトークンを選ぶ方法には、最も高い確率を選ぶ方法と、確率に従ってサンプリングする方法があります。

```python
import torch

torch.manual_seed(0)

tokens = ["atsui", "samui", "tanoshii", "isogashii", "nemui"]
probabilities = torch.tensor([0.30, 0.20, 0.15, 0.10, 0.25])

greedy_id = torch.argmax(probabilities).item()
sampled_id = torch.multinomial(probabilities, num_samples=1).item()

print("greedy:", tokens[greedy_id])
print("sampled:", tokens[sampled_id])
```

`argmax` は常に最も確率が高い候補を選びます。

`multinomial` は確率が高い候補ほど選ばれやすくしつつ、低い候補も選ばれる余地を残します。

### 11.11　確率が高いことと正しいことは違う

機械学習モデルは、確率の高い出力を返します。

しかし、確率が高いことと、正しいことは同じではありません。

これは特に大規模言語モデルを使う上で重要です。

言語モデルは、ある文脈に続きそうなトークンに高い確率を出します。

つまり、学習データのパターンから見て「もっともらしい」続きを生成します。

しかし、「もっともらしい」ことと「事実として正しい」ことは違います。

たとえば、ある人物について質問したとき、モデルが非常に自然な文章で答えたとします。

文体は自然で、構造も整っていて、いかにも正しそうに見えるかもしれません。

しかし、その内容が事実と違う可能性があります。

これは、モデルが真実そのものを直接参照しているわけではなく、学習データに基づく確率的な予測をしているからです。

```text
高確率な文章
≠
必ず正しい文章
```

特に、最新情報、専門的な事実、数値、法律、医療、金融、個別の固有名詞に関する情報では注意が必要です。

モデルが知らない情報でも、それらしく補ってしまうことがあります。

これをハルシネーションと呼ぶことがあります。

言語モデルは、確率的に自然な文章を作るのが得意です。

しかし、正確性を保証する仕組みではありません。

そのため、重要な情報を扱う場合には、検索、データベース、一次情報、検証プロセスと組み合わせる必要があります。

確率モデルとしての言語モデルを理解すると、この性質が見えやすくなります。

### 11.12　本章のまとめ

**Transformer への接続**

言語モデルとしての Transformer は、文脈を条件として次トークンの確率分布を出すモデルです。

生成時には、その確率分布から実際のトークンを選びます。つまり、Transformer の出力を理解するには、「確率が高い候補を出すこと」と「事実として正しいこと」は別である、という見方がとても重要です。

**ミニ演習**

- 第11章の PyTorch コードで、`probabilities` の値を変えて `argmax` と `multinomial` の違いを確認してみましょう。
- `P(猫 | 吾輩は)` と `P(猫)` の違いを、条件付き確率の言葉で説明してみましょう。
- 確率が高い文章と事実として正しい文章がズレる例を1つ考えてみましょう。

この章では、機械学習を確率として見る考え方を学びました。

機械学習では、不確実な問題を扱うために確率が重要になります。

分類問題では、モデルはカテゴリごとの確率を出します。

```text
犬：0.87
猫：0.13
```

言語モデルでは、次に来るトークンの確率分布を出します。

```text
猫：0.35
犬：0.08
人間：0.04
...
```

確率分布とは、複数の候補に対して、それぞれどれくらい起こりやすいかを割り当てたものです。

尤度とは、観測されたデータが、あるモデルのもとでどれくらいもっともらしいかを表す量です。

最大尤度推定では、観測されたデータが最ももっともらしくなるように、モデルのパラメータを決めます。

分類や言語モデルで使う交差エントロピー損失は、正解データの尤度を高める考え方と深くつながっています。

```text
正解の確率を高くする
=
交差エントロピー損失を小さくする
```

言語モデルは、文脈を条件として、次トークンの確率を予測します。

```text
P(次のトークン | これまでのトークン列)
```

生成時には、モデルが出した確率分布から実際のトークンを選び、それを繰り返すことで文章を作ります。

この章で一番重要な考え方は、次の一文です。

**機械学習モデルは、不確実な入力に対して候補ごとの確率を出し、学習では観測された正解データがよりもっともらしくなるようにパラメータを調整する。**

Transformer や大規模言語モデルも、この考え方の上にあります。

文脈を入力し、次トークンの確率分布を出す。  
実際に続いたトークンに高い確率を出せるように学習する。  
生成時には、その確率分布からトークンを選ぶ。

この確率的な見方を持っておくと、Transformer の出力、softmax、交差エントロピー、次トークン予測の意味がかなり理解しやすくなります。
